Phase 1 — Database & Auth (Day 1–2) · Free

1
Set up Supabase — your database + auth
Supabase is free for up to 500MB database + 50,000 monthly active users. It replaces Firebase, Auth0, and a separate Postgres server — all in one.
1
Go to supabase.com → click Start your project → sign in with GitHub → create a new project named signal-prod. Choose region: South Asia (Mumbai) for lowest latency to Indian users.
2
Once created, go to Project Settings → API → copy your Project URL and anon public key. You'll need these in every service.
3
Go to SQL Editor and run this to create your core tables:
-- Users table (extends Supabase auth.users)
CREATE TABLE profiles (
  id          uuid REFERENCES auth.users PRIMARY KEY,
  name        text,
  plan        text DEFAULT 'free',  -- free | starter | pro | elite
  plan_expiry timestamptz,
  broker      text,
  created_at  timestamptz DEFAULT now()
);

-- Stock signals
CREATE TABLE signals (
  id          bigserial PRIMARY KEY,
  symbol      text NOT NULL,
  signal_type text,  -- BUY | SELL | HOLD
  confidence  float,
  entry       float,
  target      float,
  stop_loss   float,
  indicators  jsonb,
  fired_at    timestamptz DEFAULT now(),
  outcome     text,  -- WIN | LOSS | OPEN
  exit_price  float,
  exit_at     timestamptz
);

-- User portfolios
CREATE TABLE portfolios (
  id         bigserial PRIMARY KEY,
  user_id    uuid REFERENCES profiles,
  symbol     text,
  quantity   int,
  avg_price  float,
  broker     text,
  updated_at timestamptz DEFAULT now()
);

-- Paper trading strategies
CREATE TABLE paper_strategies (
  id           bigserial PRIMARY KEY,
  user_id      uuid REFERENCES profiles,
  name         text,
  config       jsonb,  -- all strategy params
  virtual_cash float DEFAULT 100000,
  status       text DEFAULT 'running',
  started_at   timestamptz DEFAULT now()
);

-- Paper trade log
CREATE TABLE paper_trades (
  id          bigserial PRIMARY KEY,
  strategy_id bigint REFERENCES paper_strategies,
  symbol      text,
  signal      text,
  entry       float,
  exit_price  float,
  pnl         float,
  outcome     text,
  created_at  timestamptz DEFAULT now()
);

-- Broker OAuth tokens (encrypted)
CREATE TABLE broker_tokens (
  user_id      uuid REFERENCES profiles,
  broker       text,
  access_token text,  -- store AES-256 encrypted
  expires_at   timestamptz,
  PRIMARY KEY (user_id, broker)
);

-- Enable Row Level Security on all tables
ALTER TABLE profiles        ENABLE ROW LEVEL SECURITY;
ALTER TABLE portfolios      ENABLE ROW LEVEL SECURITY;
ALTER TABLE paper_strategies ENABLE ROW LEVEL SECURITY;
ALTER TABLE paper_trades    ENABLE ROW LEVEL SECURITY;
ALTER TABLE broker_tokens   ENABLE ROW LEVEL SECURITY;

-- RLS policies: users can only see their own data
CREATE POLICY "own_profile" ON profiles
  FOR ALL USING (auth.uid() = id);

CREATE POLICY "own_portfolio" ON portfolios
  FOR ALL USING (auth.uid() = user_id);
4
Go to Authentication → Providers → enable Google and Twitter/X. For Google: create OAuth credentials at console.cloud.google.com. For Twitter: create app at developer.twitter.com. Paste Client ID + Secret into Supabase. Done — auth is live.

const express = require('express');
const cors    = require('cors');
const { createClient } = require('@supabase/supabase-js');
require('dotenv').config();

const app = express();
const supabase = createClient(
  process.env.SUPABASE_URL,
  process.env.SUPABASE_SERVICE_KEY
);

app.use(cors({ origin: ['https://signal.vercel.app', 'http://localhost:3000'] }));
app.use(express.json());

// ── Auth middleware ─────────────────────────────────
async function auth(req, res, next) {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) return res.status(401).json({ error: 'Unauthorized' });
  const { data: { user }, error } = await supabase.auth.getUser(token);
  if (error || !user) return res.status(401).json({ error: 'Invalid token' });
  req.user = user;
  next();
}

// ── Routes ──────────────────────────────────────────

// GET /signals — latest signals (public)
app.get('/signals', async (req, res) => {
  const { data } = await supabase
    .from('signals')
    .select('*')
    .order('fired_at', { ascending: false })
    .limit(20);
  res.json(data);
});

// GET /portfolio — user's portfolio (auth required)
app.get('/portfolio', auth, async (req, res) => {
  const { data } = await supabase
    .from('portfolios')
    .select('*')
    .eq('user_id', req.user.id);
  res.json(data);
});

// POST /paper-trade/create — new paper strategy
app.post('/paper-trade/create', auth, async (req, res) => {
  const { name, config } = req.body;
  const { data } = await supabase
    .from('paper_strategies')
    .insert({ user_id: req.user.id, name, config })
    .select().single();
  res.json(data);
});

// GET /paper-trade/:id/log — signal log for strategy
app.get('/paper-trade/:id/log', auth, async (req, res) => {
  const { data } = await supabase
    .from('paper_trades')
    .select('*')
    .eq('strategy_id', req.params.id)
    .order('created_at', { ascending: false });
  res.json(data);
});

// POST /paper-trade/:id/update-params — edit strategy
app.post('/paper-trade/:id/update-params', auth, async (req, res) => {
  await supabase.from('paper_strategies')
    .update({ config: req.body.config })
    .eq('id', req.params.id)
    .eq('user_id', req.user.id);
  res.json({ ok: true });
});

// GET /market/quote/:symbol
app.get('/market/quote/:symbol', auth, async (req, res) => {
  // Call mStock API here
  const quote = await getMStockQuote(req.params.symbol);
  res.json(quote);
});

app.listen(process.env.PORT, () =>
  console.log(`SIGNAL API running on port ${process.env.PORT}`));

Phase 4 — ML Signal Engine (Day 5–7) · Free

4
Random Forest model on Hugging Face Spaces
Hugging Face Spaces lets you host a Python Flask/FastAPI app for free. Your ML model runs here and the backend calls it every 2 minutes during market hours.
1
Create a new Space at huggingface.co/spaces → choose Gradio or Docker → name it signal-ml.
2
Create app.py:
from flask import Flask, request, jsonify
from sklearn.ensemble import RandomForestClassifier
import pandas as pd, numpy as np, joblib
from ta import add_all_ta_features

app = Flask(__name__)
model = joblib.load('signal_rf_model.pkl')  # pre-trained model file

FEATURES = [
  'momentum_rsi', 'trend_macd', 'trend_ema_fast',
  'volatility_bbh', 'trend_adx', 'volume_obv'
]

@app.route('/predict', methods=['POST'])
def predict():
  df = pd.DataFrame(request.json['candles'],
         columns=['date','open','high','low','close','volume'])
  df = add_all_ta_features(df, open='open', high='high',
         low='low', close='close', volume='volume', fillna=True)
  proba = model.predict_proba(df[FEATURES].tail(1))[0][1]
  signal = 'BUY' if proba > 0.65 else ('SELL' if proba < 0.35 else 'HOLD')
  return jsonify({ 'signal': signal, 'confidence': round(proba * 100, 1) })

if __name__ == '__main__':
  app.run(host='0.0.0.0', port=7860)
3
Your backend calls this every 2 minutes during market hours (9:15am–3:30pm IST, Mon–Fri) using a cron job. For each NIFTY 50 stock, it fetches OHLCV from mStock → sends to ML service → stores the signal in Supabase → pushes notifications to subscribed users.
4
Training the model: Use 5 years of historical data from yfinance (free). Run training locally once, save the model file with joblib.dump(model, 'signal_rf_model.pkl'), upload to Hugging Face Space. Re-train monthly with new data.

// lib/api.ts — used by BOTH website and mobile app
const API = process.env.EXPO_PUBLIC_API_URL; // your Railway URL

export async function getSignals() {
  const r = await fetch(`${API}/signals`);
  return r.json();
}

export async function getPortfolio(token: string) {
  const r = await fetch(`${API}/portfolio`, {
    headers: { Authorization: `Bearer ${token}` }
  });
  return r.json();
}

export async function createPaperStrategy(token: string, name: string, config: object) {
  const r = await fetch(`${API}/paper-trade/create`, {
    method: 'POST',
    headers: { Authorization: `Bearer ${token}`, 'Content-Type': 'application/json' },
    body: JSON.stringify({ name, config })
  });
  return r.json();
}

Phase 6 — Payments & Subscriptions (Day 5–6) · Free to integrate

6
Razorpay subscription billing
Free to integrate. Razorpay charges 2% per transaction — only when you have paying users. Works on web and mobile.
// Backend: create subscription
const Razorpay = require('razorpay');
const rzp = new Razorpay({ key_id: process.env.RAZORPAY_KEY, key_secret: process.env.RAZORPAY_SECRET });

app.post('/subscribe', auth, async (req, res) => {
  const plans = {
    starter: 'plan_starter_monthly',  // ₹199/mo — create in Razorpay dashboard
    pro:     'plan_pro_monthly',      // ₹599/mo
    elite:   'plan_elite_monthly',    // ₹1499/mo
  };
  const sub = await rzp.subscriptions.create({
    plan_id: plans[req.body.plan],
    total_count: 12,
    notes: { user_id: req.user.id }
  });
  res.json({ payment_url: sub.short_url });
});

// Webhook: Razorpay calls this when payment succeeds
app.post('/webhook/razorpay', async (req, res) => {
  const event = req.body;
  if (event.event === 'subscription.activated') {
    const userId = event.payload.subscription.entity.notes.user_id;
    const plan   = getPlanFromPlanId(event.payload.subscription.entity.plan_id);
    // Update user's plan in Supabase
    await supabase.from('profiles').update({ plan }).eq('id', userId);
  }
  res.json({ ok: true });
});
